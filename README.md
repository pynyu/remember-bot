# 🤖 Remember Bot — нагадування та нотатки в Telegram

Телеграм-бот, який допомагає не забувати справи. Розуміє час природною мовою
українською, підтримує одноразові й повторювані нагадування, нотатки з
закріпленням, пошуком і редагуванням, а також відкладання нагадувань (snooze).

## ✨ Можливості

**Нагадування**
- Природна мова: `завтра о 9:00 зустріч`, `через 30 хвилин подзвонити`, `25.12 о 10:00 подарунки`
- Повторювані: `щодня`, `по буднях`, `щотижня у понеділок`, `щомісяця`, `щороку`
- **Повторні сповіщення** — нагадаю кілька разів поспіль, поки не натиснеш «✅ Готово»
  (рятує, коли телефон беззвучний). Кількість і інтервал — кнопками під нагадуванням.
- **Пріоритет/звук**: 🔔 звичайно / 🔇 тихо / 🔴 важливо
- Інлайн-маркери в тексті: `×3` (нагадати 3 рази), `важливо`, `тихо`
- Кнопки під сповіщенням: ✅ Готово, 😴 +10 хв / +1 год / Завтра
- 📅 Справи на сьогодні, 📊 статистика виконаного
- Часовий пояс і типові налаштування — для кожного користувача

**Нотатки**
- Будь-який текст без часу автоматично зберігається як нотатка
- Закріплення (📌), редагування, видалення з підтвердженням
- Пошук по тексту

**Технічно**
- `aiogram 3` (async), `APScheduler` (опитування БД), `SQLite`
- БД — джерело правди. Планувальник кожні 20 с перевіряє прострочені нагадування,
  тож **навіть якщо бот був вимкнений — нагадування надішлеться одразу після старту**,
  а повторювані «доганяють» розклад без дублювання.
- Схема БД мігрує автоматично при старті (нові колонки додаються без втрати даних)
- Жодних зовнішніх сервісів — усе локально

## 📁 Структура

```
bot.py              — точка входу
config.py           — читання .env
database.py         — SQLite (users, notes, reminders)
scheduler.py        — APScheduler, спрацювання нагадувань
keyboards.py        — клавіатури
states.py           — FSM-стани
handlers/           — common, reminders, notes, settings, smart (вільний текст)
utils/timeparse.py  — розпізнавання часу з тексту
utils/format.py     — форматування дат
deploy/remember-bot.service — systemd-юніт
```

## 🚀 Локальний запуск (Windows / Linux / Mac)

```bash
python -m venv .venv
# Windows:  .venv\Scripts\activate
# Linux/Mac: source .venv/bin/activate
pip install -r requirements.txt

cp .env.example .env          # Windows: copy .env.example .env
# відкрий .env і встав BOT_TOKEN від @BotFather

python bot.py
```

Токен бота отримуєш у [@BotFather](https://t.me/BotFather): `/newbot` → ім'я → username.

# 🌐 Деплой на Hetzner (де вже працює 2 боти)

Інструкція написана так, щоб **не зачепити наявні боти**. Ключова ідея:
окрема папка + окремий venv + окремий systemd-сервіс з унікальною назвою.

### 0. Підключення до сервера

```bash
ssh root@IP_СЕРВЕРА          # або твій користувач: ssh botuser@IP
```

> Якщо твої боти запущені під окремим користувачем (не root) — далі працюй під ним
> або заміни `User=` у сервіс-файлі (крок 5).

### 1. Перевір, що вже встановлено

```bash
python3 --version            # бажано 3.10+
which python3
ls /etc/systemd/system/ | grep -i bot   # побачиш наявні бот-сервіси
```

Якщо Python старий або немає `venv`:

```bash
apt update && apt install -y python3 python3-venv python3-pip
```

### 2. Створи окрему папку для бота

```bash
mkdir -p /opt/remember-bot
cd /opt/remember-bot
```

> Можна й у домашній папці, напр. `/home/botuser/remember-bot`. Тоді скрізь нижче
> заміни шлях і не забудь поправити його у `.service`-файлі.

### 3. Залий код на сервер

**Варіант А — через git** (якщо завантажиш проєкт у репозиторій):

```bash
cd /opt
git clone <URL_РЕПОЗИТОРІЮ> remember-bot
cd remember-bot
```

**Варіант Б — через scp із твого комп'ютера** (виконується ЛОКАЛЬНО, не на сервері):

```powershell
# З Windows PowerShell, з папки проєкту.
# Архівувати без .venv та .env (вони створяться на сервері).
scp -r bot.py config.py database.py scheduler.py keyboards.py states.py `
    requirements.txt .env.example handlers utils deploy `
    root@IP_СЕРВЕРА:/opt/remember-bot/
```

### 4. Створи venv та встанови залежності

```bash
cd /opt/remember-bot
python3 -m venv .venv
.venv/bin/pip install --upgrade pip
.venv/bin/pip install -r requirements.txt
```

Окремий venv = свої версії бібліотек, які не конфліктують з іншими ботами.

### 5. Налаштуй .env

```bash
cp .env.example .env
nano .env
```

Встав свій токен:

```
BOT_TOKEN=123456789:AA...твій_токен
DEFAULT_TZ=Europe/Kyiv
DB_PATH=data/bot.db
```

Збережи (`Ctrl+O`, `Enter`, `Ctrl+X`).

### 6. Перевір, що бот стартує вручну

```bash
.venv/bin/python bot.py
```

Маєш побачити `Бот запускається…`. Напиши боту в Telegram `/start` — він відповість.
Зупини `Ctrl+C`.

### 7. Налаштуй автозапуск через systemd

Скопіюй готовий юніт (унікальна назва `remember-bot` не зачепить інші сервіси):

```bash
cp /opt/remember-bot/deploy/remember-bot.service /etc/systemd/system/
```

Якщо боти крутяться під НЕ-root користувачем — відредагуй `User=` та шляхи:

```bash
nano /etc/systemd/system/remember-bot.service
```

Застосуй і запусти:

```bash
systemctl daemon-reload
systemctl enable remember-bot          # автозапуск після ребуту
systemctl start remember-bot
systemctl status remember-bot          # має бути active (running)
```

### 8. Логи та керування

```bash
journalctl -u remember-bot -f          # живі логи
systemctl restart remember-bot         # перезапуск
systemctl stop remember-bot            # зупинка
```

Інші твої боти живуть як окремі сервіси — на них ці команди не впливають.

### 9. Оновлення бота надалі

```bash
cd /opt/remember-bot
git pull                               # або заново scp змінені файли
.venv/bin/pip install -r requirements.txt   # якщо змінювались залежності
systemctl restart remember-bot
```

## 🛠 Часті проблеми

| Симптом | Причина / рішення |
|---|---|
| `BOT_TOKEN не задано` | Немає `.env` або порожній токен. Перевір крок 5. |
| `status` показує `failed` | Дивись `journalctl -u remember-bot -e`. Найчастіше — невірний шлях у `.service` або не той `User`. |
| Бот не відповідає, але сервіс `running` | Той самий токен уже запущено десь ще (один токен — один процес). Зупини дубль. |
| Нагадування не приходять | Бот був вимкнений у час спрацювання — після старту прострочені надсилаються одразу; майбутні відновлюються з БД. Перевір, що сервіс `active (running)`. |
| Час нагадувань зміщений | Постав правильний часовий пояс командою `/tz` у боті. |
| Немає звуку на iPhone | Бот не може обійти апаратний беззвучний режим iOS. Відкрий чат із ботом → тапни назву зверху → **Notifications** → увімкни звук і прибери Mute. Додатково постав більше повторів (кнопка «🔁 Повторів») — вібрація спрацює навіть у тихому режимі. |
